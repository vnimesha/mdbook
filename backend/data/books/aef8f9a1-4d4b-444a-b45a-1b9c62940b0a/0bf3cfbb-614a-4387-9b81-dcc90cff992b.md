# Lesson 1 — What is a Graph?

## The Basic Idea

A graph is a **picture of a relationship between two numbers**.

Imagine you run a lemonade stand. You track how many cups you sell and how much money you make:

| Cups sold | Money earned |
|---|---|
| 0 | $0 |
| 1 | $2 |
| 2 | $4 |
| 3 | $6 |
| 4 | $8 |

This table IS the relationship. A graph just draws this same information as a picture.

## The Two Axes

A graph has two number lines, placed at right angles:

```
  Money ($)
  |
8 |                   •
  |
6 |               •
  |
4 |           •
  |
2 |       •
  |
0 •───┬───┬───┬───┬───→ Cups sold
  0   1   2   3   4
```

- The **horizontal** line (left to right) is called the **x-axis**. Here it represents cups sold.
- The **vertical** line (bottom to top) is called the **y-axis**. Here it represents money earned.

Each dot represents one row from our table. The dot at (3, 6) means "3 cups sold, $6 earned."

## Plotting a Point

To plot the point (3, 6):

1. Start at the origin (0, 0) — the corner where the axes meet.
2. Move RIGHT along the x-axis to 3.
3. Move UP to 6.
4. Place a dot.

That's it. Every point on a graph is just a pair of numbers: (x value, y value).

## Connecting the Dots

When we connect the dots, we get a **line** or a **curve**. This visual shape tells us the story of the relationship at a glance — much faster than reading a table.

# Lesson 2 — Reading a Graph

## From Graph Back to Numbers

Reading a graph is the reverse of plotting: you look at the picture and extract numbers.

```
  y
  |
5 |           •───────•
  |          /
4 |         •
  |        /
3 |   •───•
  |  /
2 | •
  |/
1 •
  |
0 └──┬──┬──┬──┬──┬──┬──→ x
  0  1  2  3  4  5  6
```

To find the y-value when x = 2:

1. Find x = 2 on the horizontal axis.
2. Go straight up until you hit the line.
3. Go straight left to read the y-axis.
4. Answer: y = 3.

## Flat Parts, Rising Parts, Steep Parts

Look at the graph above carefully:

- From x = 0 to x = 2: the line is **rising steadily** (y goes from 1 to 3).
- From x = 2 to x = 3: the line is **flat** (y stays at 3). Nothing is changing!
- From x = 3 to x = 4: the line **rises steeply** (y jumps from 3 to 5 in one step).
- From x = 5 to x = 6: the line is **flat** again (y stays at 5).

The steeper the line, the faster y is changing. A flat line means y isn't changing at all. This concept — "how steeply is the line rising?" — is what we'll soon call the **slope**.

# Lesson 3 — Straight Lines and Slope

## What is Slope?

Slope answers one question: **"For each step I take to the right, how many steps does the line go up (or down)?"**

The formula:

```
                change in y       rise
  slope  =  ─────────────  =  ────────
                change in x       run
```

That's the whole idea. Let's make it concrete.

## Example 1: A Gentle Hill

```
  y
  |
6 |               •
  |
4 |           •
  |
2 |       •
  |
0 •───┬───┬───┬───┬──→ x
  0   1   2   3
```

Pick any two points. Let's use (0, 0) and (3, 6):

```
  slope = (6 - 0) / (3 - 0) = 6 / 3 = 2
```

This means: **for every 1 step to the right, the line goes UP by 2**. It's a fairly steep uphill.

## Example 2: A Gentle Downhill

```
  y
  |
6 •
  | \
4 |   •
  |     \
2 |       •
  |         \
0 └───┬───┬───┬───┬──→ x
  0   1   2   3
```

Using points (0, 6) and (3, 0):

```
  slope = (0 - 6) / (3 - 0) = -6 / 3 = -2
```

The slope is **negative**! This means the line goes DOWN. For every 1 step to the right, y decreases by 2.

## Example 3: A Flat Road

```
  y
  |
3 •───────────────•
  |
  |
0 └───┬───┬───┬───┬──→ x
  0   1   2   3
```

Using points (0, 3) and (3, 3):

```
  slope = (3 - 3) / (3 - 0) = 0 / 3 = 0
```

Slope is **zero**. The line is flat. Nothing is changing.

## Summary of Slope Meanings

| Slope | What it means | Picture |
|---|---|---|
| Positive (e.g., 2) | Line goes uphill | / |
| Zero | Line is flat | — |
| Negative (e.g., -3) | Line goes downhill | \ |
| Large positive (e.g., 10) | Very steep uphill | Nearly vertical / |
| Small positive (e.g., 0.1) | Barely going up | Nearly flat / |

## The Key Idea for Straight Lines

For a straight line, the slope is **the same everywhere**. It doesn't matter which two points you pick — you'll always get the same number. This is what MAKES it a straight line.

# Lesson 4 — Curves and Changing Slopes

## The Problem with Curves

Here's a curve (imagine the shape of a hill):

```
  y
  |
  |         •  •
  |       •      •
  |      •        •
  |     •           •
  |    •              •
  |   •                 •
  |  •
  | •
  |•
  └───────────────────────→ x
```

Unlike a straight line, this curve has a **different steepness at every point**. At the bottom left, it's climbing steeply. At the top, it's nearly flat. On the right side, it's going downhill.

We can't give ONE slope for the whole curve. Instead, we need to talk about the slope **at a specific point**.

## Slope at a Point: The Tangent Line

Imagine zooming in very close to one point on the curve. If you zoom in enough, the curve starts to look like a tiny straight line. That tiny straight line is called the **tangent line**, and its slope is the slope of the curve at that point.

Think of it like standing on a hillside:

- If you look at your feet, the ground looks flat-ish — it's a tiny straight patch.
- The tilt of that tiny patch is the slope where you're standing.
- On a different part of the hill, the tilt would be different.

## Visual Example: Slope at Three Points

```
  y
  |
  |              ___B___          (B: flat, slope ≈ 0)
  |            /         \
  |           /            \
  |         /                \
  |       /                    \
  |     / A                     C \
  |   /  (steep uphill,          (steep downhill,
  |  /    slope ≈ +3)              slope ≈ -3)
  | /
  └────────────────────────────────→ x
```

- **At point A:** The curve is going steeply uphill. Slope is a large positive number (say, +3).
- **At point B:** The curve is at the top and nearly flat. Slope is close to 0.
- **At point C:** The curve is going steeply downhill. Slope is a large negative number (say, -3).

Same curve, three different slopes, depending on where you look.

## Why "One Slope" Isn't Enough

For a straight line: one slope tells the whole story.
For a curve: you need a slope *for every point*. This means the slope itself becomes a **function** — it has a different value depending on where you are on the curve.

This "slope function" is exactly what a **derivative** is. Which brings us to Lesson 5.

# Lesson 5 — The Derivative (Slope at a Point)

## The Big Reveal

The **derivative** is just a fancy name for: **the slope of a curve at a specific point**.

That's it. Everything in Lesson 4 about "the slope changes depending on where you are"? The derivative makes that idea precise and computable.

## A Step-by-Step Approach (No Formulas Yet)

Let's find the slope of the curve y = x² at the point x = 3.

We can't use the rise/run formula directly (we only have one point, and we need two). So we use a trick: **pick a second point very close by, and compute the slope between them.**

y = x² means: whatever x is, square it to get y.

**Attempt 1: Use a nearby point at x = 4**

| Point | x | y = x² |
|---|---|---|
| Our point | 3 | 9 |
| Nearby point | 4 | 16 |

```
  slope ≈ (16 - 9) / (4 - 3) = 7 / 1 = 7
```

But x = 4 is a whole unit away. Let's get closer.

**Attempt 2: Use x = 3.1**

| Point | x | y = x² |
|---|---|---|
| Our point | 3 | 9 |
| Nearby point | 3.1 | 9.61 |

```
  slope ≈ (9.61 - 9) / (3.1 - 3) = 0.61 / 0.1 = 6.1
```

**Attempt 3: Use x = 3.01**

| Point | x | y = x² |
|---|---|---|
| Our point | 3 | 9 |
| Nearby point | 3.01 | 9.0601 |

```
  slope ≈ (9.0601 - 9) / (3.01 - 3) = 0.0601 / 0.01 = 6.01
```

**Attempt 4: Use x = 3.001**

| Point | x | y = x² |
|---|---|---|
| Our point | 3 | 9 |
| Nearby point | 3.001 | 9.006001 |

```
  slope ≈ (9.006001 - 9) / (3.001 - 3) = 0.006001 / 0.001 = 6.001
```

**Do you see the pattern?** As the nearby point gets closer and closer to x = 3, the slope gets closer and closer to **6**.

The derivative of x² at x = 3 is **6**.

## What We Just Did

We discovered the core idea of calculus:

1. Pick your point.
2. Pick a second point nearby.
3. Compute the slope between them (rise/run).
4. Move the second point closer. Recompute.
5. The number the slope approaches is the **derivative**.

This process is called finding the **limit**. The closer the second point gets, the more accurate our answer, and the derivative is the value we approach as the gap shrinks to zero.

## The Notation

The derivative of a function f(x) is written as:

> **f'(x)**    (read: "f prime of x")

or

> **df/dx**    (read: "d-f, d-x" or "the derivative of f with respect to x")

Both mean the same thing: "the slope of the function f at the point x."

The **d** stands for "an infinitely tiny change in." So df/dx literally means:

```
  a tiny change in f (the output)
  ─────────────────────────────── = slope at that point
  a tiny change in x (the input)
```

It's the same rise/run idea from Lesson 3, just with an infinitely tiny run.

# Lesson 6 — Derivative Rules (The Shortcuts)

## Why We Need Shortcuts

The "pick nearby points" method from Lesson 5 works, but it's slow. Mathematicians have discovered **rules** that let you find derivatives instantly, without doing all that numerical work.

You don't need to understand *why* these rules work (that requires more advanced math). Just learn to *use* them — think of them as tools in a toolbox.

## Rule 1: Constants (Flat Lines)

> If f(x) = c (any constant number), then **f'(x) = 0**

**Why:** A constant doesn't change. Its graph is a flat horizontal line. Flat = slope of zero.

- f(x) = 5 → f'(x) = 0
- f(x) = -100 → f'(x) = 0
- f(x) = 3.14 → f'(x) = 0

## Rule 2: The Power Rule

> If f(x) = xⁿ, then **f'(x) = n · xⁿ⁻¹**

In words: bring the exponent down as a multiplier, then reduce the exponent by 1.

**Examples:**

| f(x) | n | f'(x) = n · xⁿ⁻¹ | Simplified |
|---|---|---|---|
| x² | 2 | 2 · x¹ | 2x |
| x³ | 3 | 3 · x² | 3x² |
| x⁴ | 4 | 4 · x³ | 4x³ |
| x¹ (just x) | 1 | 1 · x⁰ = 1 · 1 | 1 |

**Check against our earlier work:** We found that the derivative of x² at x = 3 was 6. The power rule says f'(x) = 2x, so f'(3) = 2 × 3 = 6. It matches!

## Rule 3: Constant Multiplier

> If f(x) = c · g(x), then **f'(x) = c · g'(x)**

In words: a constant multiplier "passes through" the derivative unchanged.

**Examples:**

- f(x) = 3x² → f'(x) = 3 · (2x) = 6x
- f(x) = 5x³ → f'(x) = 5 · (3x²) = 15x²
- f(x) = 7x → f'(x) = 7 · (1) = 7

## Rule 4: Sum Rule (Add Derivatives Separately)

> If f(x) = g(x) + h(x), then **f'(x) = g'(x) + h'(x)**

In words: the derivative of a sum is the sum of the derivatives. Just do each piece separately.

**Example:** f(x) = x² + 3x + 5

Take the derivative of each term:

- x² → 2x
- 3x → 3
- 5 → 0

So: f'(x) = 2x + 3 + 0 = **2x + 3**

## Rule 5: The Exponential Function

> If f(x) = eˣ, then **f'(x) = eˣ**

This is the magical property of *e*: **it is its own derivative**. The slope of eˣ at any point x equals the value of eˣ at that same point. No other number has this property, and it's the reason *e* shows up in the sigmoid function.

## Putting Rules Together: A Full Example

Find the derivative of f(x) = 4x³ - 2x² + x - 7

Break it into pieces:

| Term | Rule used | Derivative |
|---|---|---|
| 4x³ | Power rule + constant multiplier | 4 · 3x² = 12x² |
| -2x² | Power rule + constant multiplier | -2 · 2x = -4x |
| x | Power rule (x = x¹) | 1 |
| -7 | Constant rule | 0 |

Add them up:

> f'(x) = 12x² - 4x + 1

Now we can find the slope anywhere instantly:

- At x = 0: f'(0) = 0 - 0 + 1 = **1**
- At x = 1: f'(1) = 12 - 4 + 1 = **9**
- At x = 2: f'(2) = 48 - 8 + 1 = **41**

The curve is getting steeper and steeper as x increases.

# Lesson 7 — What Derivatives Look Like on a Graph

## The Original Function vs. Its Derivative

Every function has a derivative, and that derivative is itself a function. Let's see them side by side.

### Example: f(x) = x²

The function f(x) = x² makes a U-shaped curve (a "parabola"):

```
  f(x) = x²
  y
  |
9 |  •                    •
  |
  |
4 |      •            •
  |
1 |         •      •
  |
0 |            •
  └──┬──┬──┬──┬──┬──┬──→ x
    -3 -2 -1  0  1  2  3
```

The derivative is f'(x) = 2x, which is a straight line:

```
  f'(x) = 2x   (the slope of x² at each point)
  y
  |
6 |                       •
4 |                   •
2 |               •
0 |──┬──┬──┬──•──┬──┬──┬──→ x
  -3 -2 -1  0  1  2  3
-2|           •
-4|       •
-6|   •
```

**Read these together:**

- At x = -2, the derivative is 2(-2) = **-4**. → The original curve is going downhill steeply.
- At x = 0, the derivative is 2(0) = **0**. → The original curve is flat (it's the bottom of the U).
- At x = 2, the derivative is 2(2) = **+4**. → The original curve is going uphill steeply.

The derivative graph is a **map of the slopes** of the original graph.

## Reading the Derivative Graph: A Guide

| If the derivative graph is... | Then the original function is... |
|---|---|
| Above zero (positive) | Going uphill (increasing) |
| Below zero (negative) | Going downhill (decreasing) |
| Exactly zero | Flat (a peak, valley, or flat spot) |
| Getting more positive | Getting steeper uphill |
| Getting more negative | Getting steeper downhill |

## Peaks and Valleys

A peak (hilltop) or valley (bottom of a bowl) is always where the derivative equals zero. Why? Because at the very top of a hill, you transition from going uphill to going downhill. At that transition moment, you're briefly going neither up nor down — the slope is zero.

This is extremely useful: **to find the highest or lowest point of a curve, find where the derivative equals zero.**

# Lesson 8 — Why All of This Matters for Neural Networks

## Part 1: What a Neuron Actually Does (The Big Picture)

Before any notation, let's understand what a neuron does in plain English.

A neuron is a little machine that:

1. **Takes in several numbers** (called "inputs")
2. **Weighs each input's importance** (some inputs matter more than others)
3. **Adds them all up** (plus a small adjustment called "bias")
4. **Squashes the result** into a number between 0 and 1

Here's a picture of the whole flow:

![How a neuron processes inputs](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/neuron_flow.png)

The yellow circles are inputs. The blue circle adds them up (weighted). The purple box squashes the result. You get a final output between 0 and 1.

Now let's decode every piece of notation.

---

## Part 2: Decoding the Notation

### The Master Formula

A sigmoid neuron computes:

> **output = σ(z)**    where    **z = (w₁·x₁ + w₂·x₂ + w₃·x₃ + ...) + b**

This looks scary. Let's break every symbol down.

### Symbol 1: The inputs (x₁, x₂, x₃, ...)

These are just **numbers coming into the neuron**. Like pixel brightness, or temperature, or anything you're feeding the network.

- If you have 3 inputs, you call them x₁, x₂, x₃.
- The little subscript numbers (₁, ₂, ₃) are just labels — like apartment numbers. They don't do any math.

**Example:** A neuron looking at a small image might have x₁ = 0.2, x₂ = 0.8, x₃ = 0.5 (pixel brightness values).

### Symbol 2: The weights (w₁, w₂, w₃, ...)

Each input gets its own weight. **A weight is a number that says "how important is this input?"**

- A weight of 5 means "this input matters a lot."
- A weight of 0.1 means "this input barely matters."
- A weight of -3 means "this input matters, but in a negative way (it should reduce the output)."

**Why pair them up?** Because we want to multiply each input by its weight: input x₁ gets multiplied by w₁, x₂ gets multiplied by w₂, and so on.

### Symbol 3: The bias (b)

The bias is **a single extra number added at the end**. Think of it as the neuron's "default lean" — is it predisposed to output high or low, before seeing any inputs?

- Big positive b → neuron leans toward outputting 1
- Big negative b → neuron leans toward outputting 0

### Symbol 4: z (the weighted sum)

**z is just the result of doing all the multiplication and addition**:

> z = w₁x₁ + w₂x₂ + w₃x₃ + ... + b

It's one single number. It can be negative, zero, or positive. It can be tiny or huge.

**Concrete example:** If w₁=2, w₂=-1, w₃=3, and x₁=0.5, x₂=0.8, x₃=0.2, and b=0.1:

> z = (2)(0.5) + (-1)(0.8) + (3)(0.2) + 0.1
> z = 1.0 − 0.8 + 0.6 + 0.1 = **0.9**

That's z. One number.

### Symbol 5: σ(z) — the sigmoid function

σ is a Greek letter (sigma, lowercase). It's just the **name of a function** — like f, but fancier. We could have named it f(z) — same thing.

σ is a function that takes z and squashes it into a value between 0 and 1. Its exact formula is:

> σ(z) = 1 / (1 + e⁻ᶻ)

Don't panic. Here's what it looks like graphed:

![The sigmoid function](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/sigmoid.png)

Look at what this picture tells you:

- When z is very negative (like -5), the output is **near 0**
- When z = 0, the output is **exactly 0.5**
- When z is very positive (like +5), the output is **near 1**
- In between, it's a smooth S-shape

The sigmoid **takes ANY number z and guarantees the output is between 0 and 1**. This is useful because outputs between 0 and 1 can be read as probabilities, or as "on/off" decisions.

### Symbol 6: e⁻ᶻ (the scary exponential)

This is the one piece that actually requires unpacking. Let's do it.

- **e** is a specific number, roughly 2.718. It's just a constant — like π is roughly 3.14.
- **eˣ** means "e raised to the power x." So e² = 2.718² ≈ 7.4.
- **e⁻ᶻ** means "e raised to the power negative z" — which equals 1/eᶻ.

What matters is how e⁻ᶻ *behaves*:

| If z is... | Then e⁻ᶻ is... | Then 1/(1+e⁻ᶻ) is... |
|---|---|---|
| Very negative (like -5) | Huge (≈ 148) | ≈ 1/149 ≈ **0** |
| 0 | 1 | 1/(1+1) = **0.5** |
| Very positive (like +5) | Tiny (≈ 0.007) | ≈ 1/1.007 ≈ **1** |

That's how the S-shape is made. The exponential piece is just the mathematical machinery that produces the smooth curve.

---

## Part 3: The Derivative of the Sigmoid

Now — finally — the derivative.

**The derivative of σ(z) tells us: "If I nudge z by a tiny amount, how much does the output σ(z) change?"**

This is exactly the slope at a point (Lesson 5). Let's look at the slope of σ at three different places:

![Slopes of sigmoid at different points](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/sigmoid_tangents.png)

The three straight lines are tangent lines — their slopes are the derivative of σ at those points:

- **At z = −5 (left):** The curve is nearly flat. Slope ≈ **0.007** (tiny). Nudging z here barely changes anything.
- **At z = 0 (middle):** The curve is at its steepest. Slope ≈ **0.250** (the maximum). Nudging z here has a big effect.
- **At z = +5 (right):** The curve is nearly flat again. Slope ≈ **0.007** (tiny). Nudging z here barely changes anything.

### The Derivative as Its Own Curve

If we plot σ'(z) (the slope at every z) on its own graph, we get this beautiful bell shape:

![The derivative of sigmoid](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/sigmoid_derivative.png)

This graph is powerful: **each point on this red curve tells you how sensitive the neuron is at that z value.**

- Tall red curve → steep slope → big changes possible → neuron can learn quickly
- Short red curve → flat slope → tiny changes → neuron barely responds

The neuron is most "sensitive" near z = 0 and becomes almost unresponsive when z is very positive or very negative.

---

## Part 4: Why This Matters for Learning

### What "Learning" Actually Means

When a neural network learns, it's trying to improve its output by adjusting its weights. The question it asks, billions of times, is:

> "If I nudge weight w₁ a tiny bit, does the output get better or worse?"

This is exactly a derivative question! And it's written with this notation:

> **∂output/∂w₁**

Let me decode that notation:

- **∂** is a stylized "d" — it's used instead of the regular d to signal "this is a partial derivative" (because output depends on many weights, not just w₁)
- **∂output/∂w₁** reads: "the partial derivative of output with respect to w₁"
- Plain English: **"how much does the output change when I nudge w₁ a tiny bit, while keeping everything else the same?"**

### Why Smooth Curves Are Essential

The derivative gives the network a direction to improve. If ∂output/∂w₁ = 0.5, the network knows: "increasing w₁ will increase the output." If it's -0.3, the network knows: "increasing w₁ will decrease the output."

**This only works if the curve has a meaningful slope.** Enter the disaster of the step function.

---

## Part 5: Why Perceptrons (Step Functions) Can't Learn

The step function used by perceptrons looks like this:

![The step function](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/step_function.png)

Its slope is:

- **Zero everywhere** (on the flat parts — left side and right side)
- **Infinity at z = 0** (at the cliff)

Both of these are **useless** for learning:

- **Slope of 0** tells the network "changing the weight has no effect" — the network can't tell what direction to move.
- **Slope of infinity** tells the network "any tiny change will flip the output completely" — no controlled, gradual improvement is possible.

The network is effectively **blind**. It can't figure out how to improve.

### The Sigmoid Solves This

![Step vs Sigmoid comparison](http://localhost:8000/books/aef8f9a1-4d4b-444a-b45a-1b9c62940b0a/images/comparison.png)

The sigmoid is smooth everywhere. Its slope is a nice, finite, useful number at every point. That means at any z, the network can compute ∂output/∂w₁ and know exactly which direction to adjust the weight.

**This is THE reason sigmoid neurons replaced perceptrons.** Not because sigmoid produces nicer outputs (both output 0 or 1 eventually) — but because **smooth curves have useful derivatives, and useful derivatives make learning possible.**

## Part 6: Recap — Every Symbol, One More Time

| Symbol | Plain English | Where it came from |
|---|---|---|
| x₁, x₂, ... | The inputs (numbers coming in) | Given by the data |
| w₁, w₂, ... | The weights (importance of each input) | Learned by the network |
| b | The bias (neuron's default lean) | Learned by the network |
| z | The weighted sum: w₁x₁ + w₂x₂ + ... + b | One number, can be anything |
| e | A special constant ≈ 2.718 | Math itself |
| e⁻ᶻ | The exponential that shapes the curve | Plugs into the sigmoid |
| σ(z) | The sigmoid function: 1/(1 + e⁻ᶻ) | Squashes z into 0 to 1 |
| σ'(z) | The derivative — slope of σ at point z | Tells us sensitivity |
| ∂output/∂w₁ | How much the output changes when w₁ changes a tiny bit | Tells the network how to learn |

If you understand this table, you understand how neural networks work at their core.

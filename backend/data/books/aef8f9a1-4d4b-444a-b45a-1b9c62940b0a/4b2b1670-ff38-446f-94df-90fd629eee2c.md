# Chapter 1 — Numbers, Variables, and Functions

## 1.1 What is a Variable?

In everyday life, a variable is just a **letter that stands in for a number we don't know yet** (or a number that can change).

Instead of saying:

> "Some number plus 3 equals 7"

We write:

> x + 3 = 7

Here **x** is a variable. It's a placeholder. In this case, x = 4, because 4 + 3 = 7.

Variables can be any letter: x, y, z, w, b — they're all just names for numbers.

## 1.2 Subscripts — Naming Many Variables at Once

What if we have *lots* of variables? Say we have 100 inputs to a neuron. We don't want to invent 100 different letter names. Instead, we use **subscripts**:

> x₁, x₂, x₃, ..., x₁₀₀

Read these as "x-sub-one," "x-sub-two," and so on. The little number at the bottom is just a *label*. It doesn't do any math — it just tells you *which* x you're talking about.

Think of it like apartment numbers in a building called "x":

- x₁ is apartment 1
- x₂ is apartment 2
- x₃ is apartment 3

Each apartment holds a different number.

**Example:** If x₁ = 0.5, x₂ = 0.8, and x₃ = 0.2, then these are three separate values, all living under the name "x" but distinguished by their subscript.

## 1.3 What is a Function?

A function is a **machine**: you put a number in, and you get a number out.

```
  Input → [ Machine ] → Output
```

We write this as:

> f(x) = some rule involving x

For example:

> f(x) = 2x + 1

This means: "whatever number you give me, I'll double it and add 1."

| Input (x) | Output f(x) = 2x + 1 |
|---|---|
| 0 | 1 |
| 1 | 3 |
| 2 | 5 |
| 3 | 7 |

The letter **f** is just the function's name. We could call it g, h, σ (sigma), or anything we want.

## 1.4 Evaluating a Function

"Evaluating" means plugging in a specific number.

If f(x) = x² + 1, then:

- f(3) = 3² + 1 = 9 + 1 = **10**
- f(0) = 0² + 1 = 0 + 1 = **1**
- f(-2) = (-2)² + 1 = 4 + 1 = **5**

That's it — replace every x with the number, then calculate.

## 1.5 The Number Line and Intervals

Numbers live on a line:

```
  ←──|──|──|──|──|──|──|──|──→
    -4 -3 -2 -1  0  1  2  3  4
```

When we say a value is "between 0 and 1," we mean it's somewhere on the line between those two points: 0.1, 0.5, 0.638, 0.99, etc. In the sigmoid neuron text, inputs "can take on any values between 0 and 1" — this just means the inputs are numbers in this range.

# Chapter 2 — Exponents and the Magic Number *e*

## 2.1 What are Exponents?

An exponent tells you **how many times to multiply a number by itself**.

> 2³ = 2 × 2 × 2 = 8

The small raised number (3) is the **exponent** (also called the "power"). The base number (2) is the **base**.

More examples:

- 5² = 5 × 5 = 25
- 10³ = 10 × 10 × 10 = 1000
- 7¹ = 7 (anything to the power of 1 is itself)

## 2.2 Negative Exponents

A negative exponent means "1 divided by" the positive version:

> 2⁻³ = 1 / 2³ = 1 / 8 = 0.125

General rule:

> a⁻ⁿ = 1 / aⁿ

This is crucial for understanding the sigmoid function. When you see e⁻ᶻ, it means 1/eᶻ.

## 2.3 What Makes Negative Exponents Useful?

When the exponent is very negative (a large negative number), the result gets **very close to zero**:

- 2⁻¹ = 0.5
- 2⁻⁵ = 0.03125
- 2⁻¹⁰ = 0.000977 (almost zero)
- 2⁻¹⁰⁰ ≈ 0 (essentially zero)

When the exponent is very positive, the result gets **very large**:

- 2¹⁰ = 1024
- 2²⁰ = 1,048,576

This behavior — shrinking to zero or blowing up — is exactly what makes the sigmoid function work.

## 2.4 The Special Number *e*

There is a special number in math called **e**. It equals approximately:

> e ≈ 2.71828...

It's an irrational number (the decimals go on forever), just like π ≈ 3.14159...

**Why is *e* special?** It has a magical property related to growth and calculus (which we'll see in Chapter 6). For now, just think of it as a specific number, roughly 2.718.

You can use *e* as a base for exponents just like any other number:

- e² ≈ 2.718² ≈ 7.389
- e⁻¹ ≈ 1 / 2.718 ≈ 0.368
- e⁰ = 1 (anything to the power 0 is 1)

## 2.5 The exp() Notation

Sometimes instead of writing eˣ, people write **exp(x)**. They mean exactly the same thing:

> exp(x) = eˣ

This is just a convenience — it's easier to read exp(-∑ⱼ wⱼxⱼ - b) than to write e raised to a long complicated exponent. When you see "exp" in a formula, just think "e raised to the power of whatever is in the parentheses."

## 2.6 Key Properties of eˣ to Remember

| z (input) | eᶻ (output) | e⁻ᶻ (output) |
|---|---|---|
| -100 | Almost 0 | Gigantic |
| -5 | ≈ 0.007 | ≈ 148.4 |
| 0 | 1 | 1 |
| 5 | ≈ 148.4 | ≈ 0.007 |
| 100 | Gigantic | Almost 0 |

Notice the pattern for **e⁻ᶻ** (which is what appears in the sigmoid):

- When z is large and positive → e⁻ᶻ is near **0**
- When z is large and negative → e⁻ᶻ is **huge**
- When z = 0 → e⁻ᶻ = 1

Hold onto this — it's the engine behind the sigmoid function.

# Chapter 3 — Summation Notation (The Big Sigma)

## 3.1 The Problem: Adding Up Many Things

Suppose you have five numbers and want to add them:

> a₁ + a₂ + a₃ + a₄ + a₅

That's fine for five items. But what about 1000? We need a shorthand.

## 3.2 Introducing Σ (Sigma)

The Greek capital letter sigma, **Σ**, means **"add up."** Here's how it works:

```
  5
  Σ  aⱼ   =   a₁ + a₂ + a₃ + a₄ + a₅
 j=1
```

Let's break this down piece by piece:

- **j** is the "index variable" — it's a counter that takes on each value in turn.
- **j = 1** (at the bottom) means "start counting at 1."
- **5** (at the top) means "stop counting at 5."
- **aⱼ** is what we're adding — the j-th value of a.

So the notation says: "Starting with j = 1 and going up to j = 5, add up all the aⱼ values."

## 3.3 Step-by-Step Example

Let's say w₁ = 2, w₂ = 5, w₃ = 3 and x₁ = 0.1, x₂ = 0.4, x₃ = 0.7.

Then:

```
  3
  Σ  wⱼxⱼ
 j=1
```

means:

> = w₁x₁ + w₂x₂ + w₃x₃
> = (2)(0.1) + (5)(0.4) + (3)(0.7)
> = 0.2 + 2.0 + 2.1
> = **4.3**

Each step: plug in j = 1, then j = 2, then j = 3, and add the results.

## 3.4 Why This Matters for Neural Networks

In the sigmoid neuron formula, you'll see:

> Σⱼ wⱼxⱼ

This just means: "multiply each weight by its corresponding input, then add them all up." If there are 784 inputs (like pixel values in a 28×28 image), this sum has 784 terms. The sigma notation saves us from writing them all out.

## 3.5 A Compact Notation

Sometimes the limits (top and bottom numbers) are left off when the context makes them obvious:

> Σⱼ wⱼxⱼ

This means "sum over all valid values of j" — you're expected to know from context that j runs over all the inputs.

# Chapter 4 — Vectors and the Dot Product (Linear Algebra Essentials)

## 4.1 What is a Vector?

A vector is simply an **ordered list of numbers**. That's it.

> **w** = (2, 5, 3)

This vector has three "components": the first is 2, the second is 5, the third is 3.

We often write vectors in **bold** (like **w**) or with an arrow on top (like w⃗) to distinguish them from regular single numbers (which mathematicians call "scalars").

In neural networks:

- **x** = (x₁, x₂, ..., xₙ) is the vector of all inputs
- **w** = (w₁, w₂, ..., wₙ) is the vector of all weights

Each vector just bundles up a collection of numbers into a single object.

## 4.2 The Dot Product

The **dot product** of two vectors is a single number you get by multiplying corresponding components and then adding:

> **w** · **x** = w₁x₁ + w₂x₂ + ... + wₙxₙ

That's it! It's the same summation from Chapter 3, just written in a more compact way.

**Example:**

> **w** = (2, 5, 3)
> **x** = (0.1, 0.4, 0.7)

> **w** · **x** = (2)(0.1) + (5)(0.4) + (3)(0.7) = 0.2 + 2.0 + 2.1 = **4.3**

## 4.3 Two Notations, Same Idea

These two expressions mean **exactly the same thing**:

> **w** · **x**    (dot product notation)

> Σⱼ wⱼxⱼ        (summation notation)

The sigmoid neuron text uses both. Now you know they're interchangeable.

## 4.4 What Does the Dot Product "Mean"?

Intuitively, the dot product measures **how much two vectors point in the same direction**, or more practically for neural networks, it computes a **weighted sum**. Each input xⱼ gets multiplied by its corresponding weight wⱼ, and the results are totaled.

A large dot product means the inputs align well with the weights. A dot product near zero means they don't.

# Chapter 5 — What is a Function, Really?

## 5.1 Functions Revisited

In Chapter 1 we met simple functions like f(x) = 2x + 1. Let's go deeper.

A function is any rule that takes an input and gives exactly one output. Functions can be simple or complex:

- f(x) = x² (square the input)
- f(x) = 1/x (take the reciprocal)
- f(x) = eˣ (raise e to the input's power)

## 5.2 Composing Functions (Functions Inside Functions)

You can plug one function into another. If:

> g(z) = 1 / (1 + z)

and we want to evaluate g(e⁻²), we simply replace z with e⁻² ≈ 0.135:

> g(e⁻²) = 1 / (1 + 0.135) = 1 / 1.135 ≈ 0.881

This is exactly what the sigmoid function does: it takes z, computes e⁻ᶻ, adds 1, and divides 1 by the result.

## 5.3 The Step Function

A **step function** is one of the simplest functions imaginable:

```
step(z) = { 0   if z < 0
           { 1   if z ≥ 0
```

It outputs 0 for any negative input, and 1 for any non-negative input. There's an abrupt "step" at z = 0.

If you graphed it, it would look like a staircase with one step — flat at 0 on the left, then jumping straight to 1 on the right.

This is what a perceptron uses. The problem: this jump is sudden, which makes it hard to make gradual adjustments (more on this when we discuss calculus).

## 5.4 Smooth vs. Abrupt Functions

A **smooth** function has no sudden jumps. You can draw it without lifting your pen, and it has no sharp corners. The sigmoid function is smooth. The step function is not — it has a sharp jump.

This smoothness is the entire reason we use sigmoid neurons instead of perceptrons. We'll see exactly why in Chapter 6.

# Chapter 6 — Calculus Part 1 — Derivatives (Rates of Change)

## 6.1 The Central Question of Calculus

Calculus answers one fundamental question:

> **"If I change the input a little bit, how much does the output change?"**

That's it. The whole field boils down to studying how things change.

## 6.2 Slope: The Idea Behind Derivatives

Remember straight lines from school? A line like y = 2x + 1 has a **slope** of 2. This means:

> "For every 1 unit you move to the right, the line goes up by 2 units."

Slope = rise / run = (change in output) / (change in input).

For a straight line, the slope is the same everywhere. But for a curve (like the sigmoid), the steepness changes from point to point. **A derivative is just the slope at a specific point on a curve.**

## 6.3 What is a Derivative?

The derivative of a function f(x) tells you the **instantaneous rate of change** — how fast the output is changing at any particular input value.

We write the derivative as:

> df/dx   or   f'(x)

Read df/dx as "the change in f with respect to the change in x."

**Concrete example:** Imagine you're driving a car. Your position is a function of time: position = f(time).

- Your **speed** at any instant is the derivative of position with respect to time.
- If you're going 60 km/h, then df/dt = 60 — your position changes by about 60 km for each hour of driving.

## 6.4 Derivatives of Simple Functions

Here are some basic derivative rules (you don't need to prove them — just see the pattern):

| Function f(x) | Derivative f'(x) | In words |
|---|---|---|
| 5 (a constant) | 0 | Constants don't change |
| x | 1 | x grows at rate 1 |
| 3x | 3 | 3x grows at rate 3 |
| x² | 2x | The rate of change of x² depends on where you are |
| x³ | 3x² | Same idea, more complex |
| eˣ | eˣ | e^x is its own derivative! This is why *e* is special. |

The last one is remarkable: **the derivative of eˣ is eˣ itself**. This is the "lovely property" the text mentions — it's why the sigmoid uses *e* and not some other number.

## 6.5 What Does "Small Change" Mean?

We use the symbol **Δ** (capital delta) to mean "a small change in." So:

- Δx means "a small change in x"
- Δw means "a small change in w"
- Δoutput means "a small change in the output"

If we know the derivative, we can predict changes:

> Δoutput ≈ f'(x) × Δx

In words: "The change in output is approximately the derivative times the change in input."

**Example:** If f(x) = x² and we're at x = 3:

- f'(x) = 2x, so f'(3) = 6
- If we nudge x by a tiny amount, say Δx = 0.01:
- Δoutput ≈ 6 × 0.01 = 0.06

Let's verify: f(3) = 9, and f(3.01) = 9.0601. The actual change is 0.0601, and our estimate was 0.06. Very close! The estimate gets more accurate the smaller Δx is.

## 6.6 Why Smoothness Matters for Learning

Now we arrive at the key insight for neural networks.

With a **step function** (perceptron), a tiny change in input causes either NO change in output (if you're far from the step) or a MASSIVE change (if you're right at the step). There's no middle ground. You can't make small, controlled adjustments.

With a **smooth function** (sigmoid), a small change in input ALWAYS produces a small, predictable change in output. This is exactly what equation (5) in the text is saying — and it's why sigmoid neurons are useful. You can **gradually tune** the weights and biases to improve the network's performance.

## 6.7 Linearity of Small Changes

When the text says "Δoutput is a linear function of the changes Δwⱼ and Δb," it means:

> Δoutput ≈ (something) × Δw₁ + (something) × Δw₂ + ... + (something) × Δb

Each weight change contributes independently and proportionally. Double the change in w₁, and its contribution to the output change doubles. This predictable relationship is what makes neural network training possible.

# Chapter 7 — Calculus Part 2 — Partial Derivatives

## 7.1 Functions of Multiple Variables

So far, we've looked at functions of one variable, like f(x) = x². But the output of a sigmoid neuron depends on many variables: w₁, w₂, ..., wₙ, and b.

A function of multiple variables might look like:

> f(x, y) = x² + 3xy + y²

This function takes TWO inputs (x and y) and produces one output.

**Example:** f(2, 1) = 4 + 6 + 1 = 11

## 7.2 The Problem: Which Variable Caused the Change?

If the output changes, which input was responsible? Maybe x changed. Maybe y changed. Maybe both.

A **partial derivative** answers: "How fast does the output change when I wiggle **just one** input, holding all the others fixed?"

## 7.3 The Notation

> ∂f/∂x

Read this as "the partial derivative of f with respect to x." The curly "∂" symbol (as opposed to the straight "d") tells you this is a partial derivative — there are other variables being held constant.

## 7.4 How to Compute Partial Derivatives

The rule is simple: **treat every other variable as if it were a constant number**, then take the regular derivative.

**Example:** Let f(x, y) = x² + 3xy + y²

**Finding ∂f/∂x** (treating y as a constant):

- x² → derivative is 2x (normal power rule)
- 3xy → derivative is 3y (since y is treated as a constant, 3y is just a constant multiplied by x)
- y² → derivative is 0 (it's a constant — no x involved)

> ∂f/∂x = 2x + 3y

**Finding ∂f/∂y** (treating x as a constant):

- x² → derivative is 0 (no y involved)
- 3xy → derivative is 3x (x is a constant, so derivative of 3x·y with respect to y is 3x)
- y² → derivative is 2y

> ∂f/∂y = 3x + 2y

## 7.5 What Partial Derivatives Tell You

∂f/∂x tells you: "If I increase x by a tiny amount (while keeping y fixed), how much does f change?"

∂f/∂y tells you: "If I increase y by a tiny amount (while keeping x fixed), how much does f change?"

## 7.6 The Total Change Formula

Now we can write the complete picture. If you change BOTH x and y by small amounts:

> Δf ≈ (∂f/∂x) × Δx + (∂f/∂y) × Δy

Each partial derivative acts as a "sensitivity knob" — it tells you how sensitive the output is to each particular input.

## 7.7 Connecting to Equation (5) in the Text

The formula from the text:

> Δoutput ≈ Σⱼ (∂output/∂wⱼ) × Δwⱼ + (∂output/∂b) × Δb

is just the same idea with more variables. It says:

"The total change in output equals:

- (how sensitive output is to w₁) × (change in w₁), PLUS
- (how sensitive output is to w₂) × (change in w₂), PLUS
- ... and so on for every weight, PLUS
- (how sensitive output is to b) × (change in b)"

Each partial derivative is a number telling you how much that particular weight or bias "matters" to the output. This is the foundation of how neural networks learn — you compute these sensitivities and then adjust each weight accordingly.

# Chapter 8 — Putting It All Together — The Sigmoid Neuron

Now you have every tool you need. Let's walk through the sigmoid neuron, symbol by symbol.

## 8.1 The Setup

A sigmoid neuron has:

- **Inputs:** x₁, x₂, ..., xₙ (numbers between 0 and 1)
- **Weights:** w₁, w₂, ..., wₙ (one weight per input — can be any number)
- **Bias:** b (a single number — can be any number)

## 8.2 Step 1: Compute the Weighted Sum

First, compute:

> z = **w** · **x** + b

Expanding this using what we learned:

> z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

Or in summation notation:

> z = Σⱼ wⱼxⱼ + b

**What's happening:** Each input is multiplied by its weight (measuring how important that input is), the products are added up, and then the bias is added. The result is a single number, z, which can be any real number — positive, negative, or zero.

**Worked example:** Suppose we have 3 inputs:

- Inputs: x₁ = 0.5, x₂ = 0.8, x₃ = 0.2
- Weights: w₁ = 1.0, w₂ = -2.0, w₃ = 0.5
- Bias: b = 0.3

Then:

> z = (1.0)(0.5) + (-2.0)(0.8) + (0.5)(0.2) + 0.3
> z = 0.5 + (-1.6) + 0.1 + 0.3
> z = **-0.7**

## 8.3 Step 2: Apply the Sigmoid Function

Now we feed z into the **sigmoid function**:

> σ(z) = 1 / (1 + e⁻ᶻ)

Let's compute this for z = -0.7:

1. Compute -z: -(-0.7) = 0.7
2. Compute e⁰·⁷ ≈ 2.014
3. Add 1: 1 + 2.014 = 3.014
4. Take the reciprocal: 1 / 3.014 ≈ **0.332**

The neuron's output is approximately **0.332**.

## 8.4 Step 3: Understand the Three Regimes

Let's see what happens for different values of z:

**When z is large and positive (say z = 10):**

1. e⁻¹⁰ ≈ 0.0000454 (a tiny number)
2. 1 + 0.0000454 ≈ 1.0000454
3. 1 / 1.0000454 ≈ **0.99995** (very close to 1)

**When z is large and negative (say z = -10):**

1. e⁻⁽⁻¹⁰⁾ = e¹⁰ ≈ 22026
2. 1 + 22026 = 22027
3. 1 / 22027 ≈ **0.0000454** (very close to 0)

**When z = 0:**

1. e⁰ = 1
2. 1 + 1 = 2
3. 1 / 2 = **0.5** (right in the middle)

This gives us the S-shaped curve:

```
  Output
  1.0 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ·····
                                              ···
  0.8                                     ··
                                        ·
  0.6                                 ·
                                    ·
  0.5 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─·─ ─ ─ ─ ─ ─ ─ ─ ─ (z = 0)
                                ·
  0.4                         ·
                            ·
  0.2                    ··
                     ···
  0.0 ─·····─────────── ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
      -6  -4  -2   0   2   4   6          z →
```

## 8.5 Why This Shape Matters

The sigmoid squashes ANY number into the range (0, 1):

- Huge positive inputs → output near 1
- Huge negative inputs → output near 0
- Inputs near 0 → output near 0.5

This is the "smoothed out step function" the text describes. A perceptron gives you a hard 0 or 1. The sigmoid gives you a smooth transition.

## 8.6 The Complete Formula, Decoded

The full sigmoid neuron output is:

> output = 1 / (1 + exp(-Σⱼ wⱼxⱼ - b))

Let's label every piece:

| Symbol | What it means | Chapter |
|---|---|---|
| wⱼ | The weight for input j | Ch. 1 (variables & subscripts) |
| xⱼ | The j-th input value | Ch. 1 (variables & subscripts) |
| Σⱼ | "Add up over all j" | Ch. 3 (summation) |
| wⱼxⱼ | Weight times input | Ch. 1 (multiplication) |
| Σⱼ wⱼxⱼ | Weighted sum (= **w**·**x**) | Ch. 3 & 4 (sum & dot product) |
| b | The bias term | Ch. 1 (variables) |
| Σⱼ wⱼxⱼ + b | This is z, the total input | Ch. 8.2 (combining everything) |
| exp(…) | e raised to the power of … | Ch. 2 (exponents & e) |
| e⁻ᶻ | 1/eᶻ — shrinks for large z | Ch. 2 (negative exponents) |
| 1/(1+e⁻ᶻ) | The sigmoid function σ(z) | Ch. 5 (composing functions) |
| ∂output/∂wⱼ | How sensitive output is to wⱼ | Ch. 7 (partial derivatives) |
| Δwⱼ | A small change in weight j | Ch. 6 (small changes) |
| Δoutput | The resulting change in output | Ch. 6 (derivatives) |

## 8.7 The Key Equation for Learning

> Δoutput ≈ Σⱼ (∂output/∂wⱼ) × Δwⱼ + (∂output/∂b) × Δb

In plain English:

"If you make small tweaks to the weights and bias, you can predict exactly how the output will change. Each partial derivative tells you the 'sensitivity' of the output to that particular knob. Multiply each sensitivity by the size of the tweak, add them all up, and you get the total change."

This is why smoothness matters: because the sigmoid is smooth, these partial derivatives exist everywhere and change gradually. You can always compute them and use them to improve the network, one small step at a time.

## 8.8 Interpreting the Output

The output of a sigmoid neuron is a number between 0 and 1. You can interpret it as:

- A **probability** ("I'm 83% confident this is a 9")
- An **intensity** ("This pixel has 33% brightness")
- A **decision**, with a threshold: if output ≥ 0.5, call it "yes"; otherwise "no"

The choice of interpretation depends on the problem.
